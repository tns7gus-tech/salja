- `SALJA_MASTER_PLAN.md`를 실제 화면 설계로 전환하기 위한 빠른 제작 문서
- 디자이너/개발자가 바로 공통 언어로 사용할 수 있는 레이아웃, 컴포넌트, 인터랙션 기준 제공

## 2. 정보 구조 (IA)
1. Header
2. Hero (Ticker 강조)
3. Live Card (5요소 콘텐츠)

## 5. 핵심 컴포넌트 정의
1. `Header`
- 좌측 고정 로고: `SALJA`
- 우측 언어 버튼: `ko/en/ja/es`

2. `Hero`
- 대형 타이포 `SALJA` (가장 큰 시각요소)
- 서브카피 + 1차 CTA

3. `SaljaCard`
- `Ticker: SALJA` 배지
- 5개 슬롯 고정
- 저장/공유 액션 버튼

4. `LanguageMenu`
- 버튼 클릭 시 목록 열림
- 언어 변경 즉시 UI 텍스트 + 콘텐츠 교체

5. `SafetyPanel`
- 비의료 서비스 고지
- 위기 대응 문구 및 리소스 안내 영역

## 6. 인터랙션 시나리오
1. 첫 방문
- 브라우저 언어 감지
- 지원 언어면 자동 설정, 아니면 `ko` 기본

2. `오늘의 SALJA 받기`
- 5요소 콘텐츠 즉시 생성/렌더링

3. `다시 받기`
- 동일 구조로 새 콘텐츠 재생성

4. `문장 저장`
- 카드 텍스트를 클립보드 복사

5. 언어 변경
- 버튼 선택 즉시 전체 UI 라벨 전환
- 콘텐츠도 해당 언어로 재생성

## 7. 카피 길이 규칙
- Meme Caption: 12~60자
- Comfort Line: 20~90자
- Short Poem: 2~4행, 총 120자 이내
- Reframed Quote: 20~100자
- One Small Action: 10~60자

## 8. 접근성/반응형 최소 기준
- 텍스트 대비 4.5:1 이상
- 키보드 포커스 표시
- 모바일 360px 이상 대응
- 버튼 터치 영역 최소 40px

## 9. 개발 즉시 체크포인트
- `SALJA` 타이포가 첫 화면에서 가장 크게 보이는가
- 언어 전환 후 카드/버튼 문구가 모두 바뀌는가
- 생성/다시받기/저장 액션이 모두 동작하는가
- 모바일에서 레이아웃 깨짐이 없는가

## 10. 구현 파일 맵 (MVP)
- `index.html`: 페이지 구조(레이아웃/섹션/요소)
- `styles.css`: 시각 스타일, 반응형, 애니메이션
- `app.js`: 다국어 전환, 콘텐츠 생성, 저장/공유 동작
- `SALJA_MASTER_PLAN.md`: 총괄 기획
- `SALJA_WIREFRAME.md`: 와이어프레임/컴포넌트/동작 정의
- `DEPLOY.md`: Vercel/Netlify 배포 가이드
